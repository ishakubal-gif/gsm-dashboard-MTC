Applied MTC Concepts

This Decision Engine is mathematically evaluating real-time telemetry datasets using core concepts heavily drawn from Mathematics of Telecom and Computing (MTC). Below is an academic breakdown of how probability and statistical models actively drive our recommendation outputs.

📊

Mean & Variance Analysis

What it is: Mathematical measures of central tendency and statistical data spread across a distribution.

Where it's used: Applied continuously to the incoming RSSI dataset to calculate the baseline signal power (Mean) at specific campus zones and compute stability / fluctuations (Variance).

Output: Directly isolates campus zones exhibiting wildly fluctuating or chronically poor signal integrity, fundamentally driving priority rankings.

🎲

Probability Analysis

What it is: Calculating the likelihood of a specific event occurring (e.g., a call dropping) out of a random experiment (telemetry test).

Where it's used: Computes the experimental probability of call failure specifically isolating connection drops against successful pings across diverse campus coordinates.

Output: Yields the mathematical 'Drop Rate' multiplier. A sudden spike objectively supersedes pure RSSI limits to alert on critical zone failures.

🔌

Signal Decay Modeling

What it is: Contextualizing wireless telecommunications strength mathematically via logarithmic representation (dBm).

Where it's used: Signal is inherently non-linear. By treating RSSI via heuristic threshold distributions (-40 to -125), we effectively model genuine physical transmission characteristics.

Output: Generates the definitive Weak, Medium, and Strong boundaries mapping our entire UI visual classification logic.

📐

Feature Vector Processing

What it is: Abstracting real-world field observations into standardized multi-dimensional data arrays.

Where it's used: Each user report maps into a feature vector dataset array: [Location, RSSI, CallResult]. This strictly dictates how the architecture filters and queries the database.

Output: Fuels the comparative algorithm assigning numerical Priority Scores, yielding our exact final answer for infrastructure extension.
